Basic Git Concepts

Key Terms

Repository (Repo)
A project folder that Git tracks. Think of it as your project's home.
Commit
A snapshot of your code at a specific point in time. Like saving your game progress.
Branch
A separate line of development. Like creating a copy of your project to try something new.
Clone
Making a copy of a repository on your computer.
Push
Sending your changes to a remote repository (like GitHub).
Pull
Getting the latest changes from a remote repository.

The Three States of Git

Working Directory

Where you make changes to your files

  • Files you're currently editing
  • Changes not yet tracked by Git

Staging Area

Where you prepare changes for a commit

  • Changes you want to include in your next save point
  • Like choosing which photos to include in an album

Repository

Where Git stores your commit history

  • Your project's complete history
  • All save points you've made

Basic Workflow

  1. Make Changes - Edit your files in VS Code
  2. Stage Changes - Choose which changes to include in your commit
  3. Commit Changes - Save your changes with a descriptive message
  4. Push Changes - Share your changes with others (optional)
