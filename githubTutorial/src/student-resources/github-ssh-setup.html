GitHub SSH Key Setup

Set up secure authentication between your computer and GitHub using SSH keys.

What are SSH Keys?

SSH keys provide a secure way to authenticate with GitHub without entering your password every time. Think of it as a special digital key that proves you are who you say you are.

Setup Steps

Step 1: Generate SSH Key

Open your terminal and run this command:

ssh-keygen -t ed25519 -C "your_email@example.com"

Note: Replace your_email@example.com with your actual GitHub email address.

When prompted:

  • Press Enter to accept the default file location
  • Optionally, enter a passphrase for extra security (or press Enter to skip)

Step 2: Copy Your SSH Key

Display your public key with this command:

cat ~/.ssh/id_ed25519.pub

Copy the entire output (it should start with ssh-ed25519)

Step 3: Add Key to GitHub

  1. Go to GitHub → Settings → SSH and GPG keys
  2. Click "New SSH key"
  3. Give it a title (e.g., "My Laptop")
  4. Paste your SSH key into the "Key" field
  5. Click "Add SSH key"

Step 4: Test Your Connection

Verify that everything works:

ssh -T git@github.com

You should see a message like: "Hi username! You've successfully authenticated..."

Using SSH with GitHub

Clone with SSH

When cloning repositories, use the SSH URL:

git clone git@github.com:username/repository.git

Change Existing Repo to SSH

If you already cloned with HTTPS, switch to SSH:

git remote set-url origin git@github.com:username/repository.git

Troubleshooting

Permission Denied Error

If you get "Permission denied", make sure:

  • You copied the entire SSH key correctly
  • You added the key to the correct GitHub account
  • The key is in the default location (~/.ssh/id_ed25519.pub)

Multiple SSH Keys

To list all your SSH keys:

ls -la ~/.ssh
