Frequently Asked Questions

Find answers to common questions about Git, GitHub, and VS Code.

How do I undo my last commit?

If you haven't pushed the commit:

git reset --soft HEAD~1

This will undo the commit but keep your changes staged. If you want to completely remove the changes:

git reset --hard HEAD~1

Note: Be careful with --hard as it permanently deletes changes!

What's the difference between git fetch and git pull?

git fetch downloads changes from the remote repository but doesn't merge them into your working files. It's like checking what's new.

git pull does both: it fetches AND merges the changes into your current branch. It's like fetch + merge in one step.

How do I resolve merge conflicts?

When Git can't automatically merge changes, follow these steps:

  1. Open the conflicted files (marked with <<<<<<< markers)
  2. Choose which changes to keep (everything between <<<<<<< and >>>>>>>)
  3. Remove the conflict markers
  4. Save the file
  5. Stage and commit the resolved files

Why can't I push my changes?

Common reasons include:

  • Remote has changes you don't have locally (Solution: git pull first)
  • No upstream branch set (Solution: git push -u origin branch-name)
  • No write access to repository (Solution: Check permissions)

How do I switch to a different branch?

Use:

git checkout branch-name

Or for newer Git versions:

git switch branch-name

To create and switch to a new branch in one command:

git checkout -b new-branch-name

How do I update my forked repository?

To sync your fork with the original repository:

# Add the original repo as upstream
git remote add upstream original-repo-url

# Fetch changes
git fetch upstream

# Merge changes into your main branch
git checkout main
git merge upstream/main

What should I put in my .gitignore file?

Common entries include:

# Dependencies
node_modules/
venv/

# Build outputs
dist/
build/

# IDE settings
.vscode/
.idea/

# Environment variables
.env
.env.local

# System files
.DS_Store
Thumbs.db

How do I revert to a specific commit?

To create a new commit that undoes a previous commit:

git revert commit-hash

To actually go back in time (dangerous with shared repositories):

git reset --hard commit-hash

Why use branches?

Branches are useful for:

  • Developing features without affecting the main code
  • Multiple people working on different features simultaneously
  • Testing experimental changes safely
  • Creating pull requests for code review

How do I fix "fatal: refusing to merge unrelated histories"?

This happens when trying to merge repositories with no common commit history. To force the merge:

git pull origin main --allow-unrelated-histories

Note: Only use this if you're sure you want to combine the histories!
